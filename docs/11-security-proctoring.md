# 11. Security and Academic Integrity

## 11.1 Security Baseline

- JWT authentication protects API access.
- Backend role checks protect privileged endpoints.
- Ownership checks limit staff access to assigned offerings and owned/assigned exams.
- Students see only eligible published exams.
- Passwords are stored with secure hashing.
- API errors are returned through a consistent error response format.

## 11.2 Exam Entry Controls

- Published exams can require access-code or staff approval before attempt start.
- Entry codes are short-lived and generated by staff from exam details/live monitor.
- Staff can allow, reject, revoke, or remove access.
- Rules are shown before the timer starts.
- Device-change requests are visible to staff.

## 11.3 Integrity Tracking

The current system captures and displays baseline integrity signals:

- Fullscreen state changes.
- Blur/tab-switch style events.
- Device/camera status signals from frontend hooks.
- Attempt heartbeat.
- Auto-submit policy state.
- Live integrity stream in monitor and review screens.

These controls improve visibility and auditability but do not replace a dedicated secure browser.

## 11.4 Technical Answer Safety

SQL and C# Run features are safe previews. They validate structure and intent but do not execute arbitrary student code directly in the main backend or against the application database.

Production execution must use an isolated sandbox with:

- CPU and memory limits.
- Strict timeout.
- No internet access.
- No server filesystem access.
- No environment variable exposure.
- No access to the application database.
- Read-only or per-attempt SQL datasets.
- Output size limits.

## 11.5 Auditability

The system records audit events for important actions such as login-sensitive workflows, admin changes, access-code generation, approvals, integrity events, grading, and result publication.

## 11.6 Practical Limitation

Browser-based integrity controls cannot fully prevent external-device recording or all forms of cheating. The project focuses on realistic controls: eligibility, controlled entry, visible monitoring, audit logs, and professor review.
