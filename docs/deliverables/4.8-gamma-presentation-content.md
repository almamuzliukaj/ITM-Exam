# 4.8 Sllajdet e prezantimit - Material per Gamma

Ky dokument mund te perdoret si material baze per Gamma. Mund ta kopjoni seksionin "Prompt per Gamma" dhe pastaj te kontrolloni/ndryshoni sllajdet qe gjenerohen.

## Prompt per Gamma

Create a clear, professional university project presentation for a software engineering project named "ITM Exam - University Online Examination System".

The presentation should be in Albanian and should have a clean academic/institutional style, not too decorative. Use concise slides, professional wording, and a structure suitable for a final university project presentation.

Include these slides:

1. Title slide with project name and team members:
   - Agnesa Maxhuni
   - Albiona Maxhuni
   - Alma Muzliukaj

2. Problem addressed:
   Explain that online exams in universities require better organization, secure access, question management, student eligibility, grading, result publication, and monitoring.

3. Proposed solution:
   Present ITM Exam as a role-based platform for Admin, Professor, Assistant, and Student.

4. Target users:
   Admins manage academic structure and users; professors/assistants create and grade exams; students take exams and view published results.

5. Technologies used:
   React + Vite frontend, ASP.NET Core Web API backend, PostgreSQL database, Entity Framework Core, JWT authentication, Railway backend/database deployment, Vercel frontend deployment.

6. Main modules:
   Authentication, user management, terms/courses/offerings, enrollments, question bank, exam creation, student exam session, integrity violations, gradebook, results, reports/export.

7. Student exam flow:
   Student sees available exams, reads rules, starts exam, answers questions, autosaves/submits, and later sees published results.

8. Professor/admin workflow:
   Professor creates exams/questions, reviews attempts, grades answers, publishes results. Admin manages users, academic structure, enrollments, and reports.

9. Security and integrity:
   Role-based access, JWT, eligibility checks, access control, violation logging, fullscreen/session monitoring, audit logs.

10. Main development challenges:
   Role-based authorization, connecting academic structure with exams, ensuring students only see eligible exams, grading workflows, deployment configuration, i18n, and merge/build fixes.

11. Results achieved:
   Functional full-stack system with real backend, database, frontend dashboards, exam workflow, question bank, grading, results, reporting, and deployment readiness.

12. Future work:
   Improve live monitoring, add stronger proctoring integrations, expand AI-assisted grading/question generation, add more automated tests, improve analytics, and integrate with official university systems.

Keep each slide short and visually clean. Use bullet points, not long paragraphs. Use a professional blue/white academic color palette.
